Contexte
Nous avons un prototype fonctionnel du site utiles67 (mobile-first, feed de mesures, fiches, visages UTILES, agenda, rejoindre, CTA).
Nous voulons maintenant transformer ce prototype en site réel, propre, maintenable et déployable en production.

Objectif
Livrer un site statique PRODUCTION READY, déployable sur Cloudflare Pages via Git (repo source-of-truth),
avec contenus pilotés par fichiers JSON et images gérées proprement.
Aucun chatbot pour l’instant : les questions passent par un humain (formulaire).

IMPORTANT (contrat de prod)
- Site statique (HTML/CSS/JS uniquement)
- Aucune dépendance serveur obligatoire
- Tout doit fonctionner avec des chemins relatifs
- Le site doit pouvoir être déployé tel quel sur Cloudflare Pages
- Code propre, structuré, lisible, maintenable
- Performances mobile prioritaires

# 3) IMAGES (OBLIGATOIRE)

- Intégrer de vraies images temporaires dans le design (hero + covers + profils) pour valider le rendu.
- Ajouter des placeholders élégants si image manquante.

Règles :
- Toutes les images doivent être dans /assets/*
- coverImage pour les cartes mesures (bandeau 80–120px mobile)
- heroImage en haut de la home ("Le Strasbourg qu’on aime.")
- photo profil grande dans la modale (cover 160–220px mobile ou grand portrait)

Performance :
- utiliser loading="lazy" sur images non critiques
- dimensionner correctement les images (ne pas charger une image 4000px)
- prévoir alt text

# 5) COLLECTE DES ACTIONS (SANS BACKEND OBLIGATOIRE)

Pas de chatbot pour l’instant.

Nous voulons :
A) "Posez votre question" (humain)
- bouton clair en home + header ou section dédiée
- ouvre une modale formulaire courte : nom (optionnel), email (recommandé), message, quartier (optionnel)
- envoi vers une URL configurable

B) "Rejoindre le mouvement"
- page dédiée avec formulaire + numéro de téléphone visible
- envoi vers URL configurable

C) "Soutien / Aider / Commentaire" sur les mesures
- envoi vers URL configurable

Implémentation technique :
- créer un fichier /js/config.js (ou objet CONFIG dans app.js) :
  CONFIG.FORM_JOIN_URL
  CONFIG.FORM_QUESTION_URL
  CONFIG.FORM_ENGAGEMENT_URL
  CONFIG.PHONE_NUMBER
- si URL non définie : fallback localStorage + message “merci”
- si URL définie : POST JSON vers cette URL

Stockage local (fallback) :
- conserver les contributions dans localStorage (liste)
- pas d’affichage public
